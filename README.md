# NV-Disruptron (LifeLine Grid)

Agentic London transport disruption intelligence for **NVIDIA Hack for Impact London 2026** (Urban Operations track).

**Fully local-first — no cloud API keys required.**

## Quick start (zero keys)

```bash
# 1. Prepare ward/IMD data (one-time)
cd london-spatial-mcp && uv sync && uv run python ../scripts/prepare_data.py

# 2. Start local Nemotron Omni (vLLM NVFP4 on :8000)
./scripts/start_vllm_backend.sh

# 3. Run AI-Q + MCP tools
cd /home/nvidia/aiq && ./scripts/start_cli.sh
# Uses configs/config_cli_default.yml → local vLLM + London MCP

# Or smoke test non-interactively:
./scripts/test_aiq_lifeline.sh
```

## No cloud API keys

| Component | Key? | Source |
|-----------|------|--------|
| Nemotron inference | **No** | Local vLLM — [NVFP4 on HuggingFace](https://huggingface.co/nvidia/Nemotron-3-Nano-Omni-30B-A3B-Reasoning-NVFP4) |
| AI-Q agents | **No** | `http://127.0.0.1:8000/v1` via `_type: nim` |
| TfL live data | **No** | Anonymous TfL API (~50 req/min) |
| London IMD / wards | **No** | London Datastore CSV (bundled) |
| postcodes.io | **No** | Free ward geocoding |

Optional only:
- `TFL_APP_KEY` — higher TfL rate limit (500 req/min)
- `HF_TOKEN` — only if HuggingFace model download is gated

NVIDIA Build **"Generate API Key"** is for hosted cloud NIM only — not used when self-hosting. See [Self-Hosted Deployments](https://build.nvidia.com/nvidia/nemotron-3-nano-omni-30b-a3b-reasoning) and [DGX Spark vLLM](https://build.nvidia.com/spark/vllm).

## MCP servers

| Server | Path | Tools |
|--------|------|-------|
| **TfL London** | `tfl-mcp-server/` | Tube/road disruptions, live traffic, air quality, stops, arrivals, journeys |
| **London Spatial** | `london-spatial-mcp/` | Ward lookup, IMD deprivation, postcode/coordinate resolution |
| **London Impact** | `london-impact-mcp/` | Line→ward mapping, disruption impact scoring |

## AI-Q configs

| Config | Use |
|--------|-----|
| `configs/config_urban_ops.yml` | **Primary** — full pipeline + MCP + local vLLM |
| `configs/config_lifeline_shallow_vllm.yml` | Shallow-only `nat run` tests |
| `aiq/configs/config_cli_default.yml` | Same as urban_ops (default for `start_cli.sh`) |

## Local inference

```bash
./scripts/start_vllm_backend.sh      # vLLM NVFP4 (recommended)
./scripts/start_nemotron_backend.sh  # fallback: llama.cpp GGUF
./scripts/test_nemotron_backend.sh   # smoke test :8000
./scripts/start_lifeline.sh          # vLLM + AI-Q CLI together
```

| Setting | Value |
|---------|-------|
| Model | `nvidia/Nemotron-3-Nano-Omni-30B-A3B-Reasoning-NVFP4` |
| Served name | `nemotron_3_nano_omni` |
| Endpoint | `http://127.0.0.1:8000/v1` |

## Example query

> "The Jubilee line has delays. Which deprived wards along the route are most affected, and what's the estimated economic exposure?"

Agents chain: `tfl_london__get_line_status` → `london_impact__score_line_disruption_impact` → `london_spatial__rank_most_deprived_wards`.
